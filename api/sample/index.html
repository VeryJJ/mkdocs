Dubbo接口名称

功能介绍

对接口提供的功能进行介绍

接入方可根据参数中的检索策略获取不同条件下的模板信息，其中特性值为最主要的查询范围的收敛条件；特性值之间为且(AND)的关系，即：同时满足列举的特性值集合中的条件；当查询结果为多条数据，需要业务方自身过滤或增加特性值继续收敛到一条为止。

Dubbo接口声明

package cn.gov.zcy.paas.template.api.service;

public interface TemplateReadService {

    Response<List<SearchTemplate>> listTemplateBy(@Valid @NotNull(message = ErrorMsg.SEARCH_PARAM_CANNOT_NULL) SearchParam searchParam);

}

请求参数说明
入参名称 类型 是否必填 值域范围 描述/示例
searchType SearchTypeEnum Y BY_RULE_IGNORE_DEFAULT(0, "忽略规则特性搜索-默认模板文件")
BY_RULE_DEFAULT(1, "按规则特性搜索-默认模板文件")
BY_RULE_VERSION(2, "按规则特性搜索-指定版本号文件")
BY_RULE_LATEST(3, "按规则特性搜索-最新版本文件")
 默认BY_RULE_IGNORE_DEFAULT
rules List N N/A 特性值集合，特性编码非必填，特性值必填
rules.code String N 字符串(1-50) 特性编码
rules.value String Y 字符串(1-50) 特性值
rules.generalized Boolean N true, false 是否开启向后通配，即支持通配符匹配，针对树状结构；
默认关闭；
eg: 339900 开启后 * 即可匹配到；否则精确匹配，只能匹配到339900；
templateType TemplateTypeEnum Y 详见TemplateTypeEnum枚举，持续增长
userCategory String Y 用户类别，参考： cn.gov.zcy.paas.user.enums.UserTypeEnum 适用场景： cn.gov.zcy.paas.template.api.enums.SearchTypeEnum.BY_RULE
distCode String N 继承区划管理编码规则 模板中心不校验区划是否合法（禁用、删除、任意值等非法值），校验工作由业务方自行保证
orgId Long N 继承机构业务的编码规则 模板中心不校验机构是否合法（禁用、删除、任意值等非法值），校验工作由业务方自行保证
majorVersion Integer N 从0开始 结合searchType参数使用，不传时默认为0
minorVersion Integer N 从0开始 结合searchType参数使用，不传时默认为0
queryRule Boolean N true, false 是否查询模板关联的特性
queryPageOfficeCode Boolean N true, false 是否查询转换后适应pageOffice的书签编码

代码示例

建议把此API的各种玩法示例都写全，防止业务方不会用、用错。

/**
 * 根据特性查询模板的dubbo接口
 */
@Test
public void searchTemplateByRuleTest() {

    SearchParam.RuleSearchParam ruleSearchParam = new SearchParam.RuleSearchParam();
    ruleSearchParam.setValue("部门集中采购-委托本级部门");
    ruleSearchParam = new SearchParam.RuleSearchParam();
    ruleSearchParam.setValue("A0101");
    ruleSearchParam.setGeneralized(true);

    List<SearchParam.RuleSearchParam> rules = new ArrayList<>();
    rules.add(ruleSearchParam);

    SearchParam searchParam = new SearchParam();
    searchParam.setRules(rules);
    searchParam.setDistCode("339900");
    searchParam.setOrgId(-1L);
    searchParam.setUserCategory("88");
    searchParam.setTemplateType(TemplateTypeEnum.CONTRACT);
    searchParam.setSearchType(SearchTypeEnum.BY_RULE_IGNORE_DEFAULT);

    Response<List<SearchTemplate>> resp = templateReadService.listTemplateBy(searchParam);

    System.out.println(resp);
}
